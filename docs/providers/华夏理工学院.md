```json

  "providerId": "whhxit",
  "eduSystemType": "zf_jwglxt",

  "authConfig": {
    "captchaRequired": false,
    "captchaKind": "none",
    "uiPreset": "password",
    "passwordTransform": "rsa_public_key"
  },
  "capabilities": {
    "course": true,
    "score": false,
    "exam": false,
    "profile": true
  }

```

说明：

- 当前验证范围只覆盖登录、学生基本信息和课表。
- 当前登录属于 `direct_password`，教务系统类型是 `zf_jwglxt`。这两个字段不要混用：其他正方学校如果有普通图片验证码，应改为 `password_captcha` 并设置 `authConfig.captchaRequired=true`；华夏理工当前为 `captchaRequired=false`。
- 后端实现时仍需复刻正方 `jwglxt` 的真实表单提交、CSRF 和前端 RSA 密码处理逻辑。
- 课表不是 EAMS `TaskActivity` 脚本，而是正方 `jwglxt` JSON。
- 首页可见学生姓名、学院、班级、当前学期日历、课表小组件、消息、成绩评价摘要。

## 登录流程

登录页：

```text
GET /jwglxt/xtgl/login_slogin.html
```

首页登录后地址：

```text
/jwglxt/xtgl/index_initMenu.html?jsdm=xs&_t={timestamp}&echarts=1
```

登录表单字段：

```text
csrftoken  hidden
language   hidden，默认 zh_CN
ydType     hidden
yhm        用户名
mm         密码输入框/提交字段
```

关键点：

- 站点会重定向到 `/jwglxt/xtgl/login_slogin.html`。
- 页面标题为 `教学管理信息服务平台`。
- 登录按钮 id 为 `dl`，文本为 `登录`。
- 直接操作 DOM 设置 `#yhm` 和 `#mm` 后点击 `#dl` 可以登录成功。
- 不需要图形验证码。
- 登录成功后可通过是否进入 `index_initMenu.html?jsdm=xs` 判断 Session 有效。

## 课表流程

课表页面：

```text
GET /jwglxt/kbcx/xskbcx_cxXskbcxIndex.html?gnmkdm=N2151
```

课表 JSON 接口：

```text
POST /jwglxt/kbcx/xskbcx_cxXsgrkb.html?gnmkdm=N2151
Content-Type: application/x-www-form-urlencoded;charset=UTF-8
X-Requested-With: XMLHttpRequest
```

最小有效参数：

```text
xnm=2025
xqm=3
```

学期参数：

```text
xqm=3   第 1 学期
xqm=12  第 2 学期
xqm=16  第 3 学期
```

页面默认选项示例：

```text
xnm=2026
xqm=3
```

但验证时当前首页日历显示的是：

```text
2025-2026 学年 2 学期
2026-02-23 至 2026-08-07
```

验证结果：

```text
2025-2026 第 1 学期: kbList 9 条，sjkList 2 条
2025-2026 第 2 学期: kbList 0 条，sjkList 4 条
2026-2027 第 1 学期: kbList 0 条，sjkList 0 条
```

因此适配时不要只看 `kbList`，也要处理 `sjkList`。实践课、课外课、网络课可能只出现在 `sjkList`。

## 解析建议

统一输出到现有 Course 模型时：

```ts
interface Course {
  id?: string
  name: string
  teacher?: string
  classroom?: string
  weekday?: number
  startSection?: number
  endSection?: number
  weeks: number[]
  rawWeeks?: string
  campus?: string
  remark?: string
  source?: unknown
}
```

映射规则：

- `kbList[].kcmc` -> `name`
- `kbList[].xm` -> `teacher`
- `kbList[].cdmc` 或 `lh + cdmc` -> `classroom`
- `kbList[].xqj` -> `weekday`
- `kbList[].jcor` / `jcs` -> `startSection`、`endSection`
- `kbList[].zcd` -> `rawWeeks`，解析成 `weeks`
- `kbList[].xqmc` -> `campus`
- `sjkList` 中没有明确星期和节次时，不要强行填充；保留 `rawWeeks` 和 `remark`

## 已知风险

- 正方 `jwglxt` 登录脚本可能对密码字段做前端处理，服务端实现时要抓真实 POST 行为，不要只照抄 Firecrawl DOM 操作。
- 学年学期默认值不一定等于首页当前学期，需要从课表页选项、首页学期文本或后端返回共同判断。
- `kbList` 为空不代表没有课，需要继续读取 `sjkList`。
- JSON 字段很多，先只做课表必要字段映射，原始对象放入 `source` 便于排查。
- 日志禁止输出账号、密码、Cookie、csrftoken。
