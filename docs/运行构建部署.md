# 运行构建部署

## 数据库初始化

```bash
cd backend
cp .env.example .env
pnpm install
pnpm prisma generate
pnpm prisma db push
pnpm seed:schools
```

## 数据库更新

涉及 Prisma schema 或数据库结构更新时，默认用这个，保留本地数据：

```bash
cd backend
pnpm prisma db push
pnpm prisma generate
pnpm build
```

不要直接用 `pnpm prisma migrate dev`。如果提示 drift、reset、All data will be lost，直接取消，改用上面的 `db push`。

只有在空数据库或确认可以重建数据库时，才用：

```bash
cd backend
pnpm prisma migrate dev
```

如果 `prisma generate` 报 `EPERM query_engine-windows.dll.node`，先关闭后端服务、Prisma Studio、微信开发者工具，再重跑。

## 查看本地数据库

当前没有云数据库，默认看本地库：

```bash
cd backend
pnpm prisma studio
```

## 后端运行

```bash
cd backend
pnpm install
pnpm prisma generate
pnpm build
pnpm dev
```

## 后端更新

```bash
cd backend
pnpm install
pnpm prisma generate
pnpm build
pnpm start
```

## 前端运行

```bash
cd frontend
pnpm install
pnpm dev:weapp
```

## 前端更新

```bash
cd frontend
pnpm install
pnpm build:weapp
```

## 后端地址

前端统一使用 `TARO_APP_API_BASE_URL`。

默认本地：

```bash
http://localhost:3000/api/v1
```

生产地址先写注释，等云数据库和生产后端配置好再启用。

## 类似功能更新要求

如果涉及用户导入、绑定、同步：

- 平台 ID 必须对应“某个学校的某个学生”。
- 后端用 `schoolId + providerId + 学号` 识别同一个学生。
- 不保存明文学号，用 `studentNoHash`。
- 重复导入必须复用同一个 `bindingId`。
- 如果先生成临时 `bindingId`，解析到学号后要合并到稳定 `bindingId`。
- 前端收到后端返回的新 `bindingId` 后，要更新本地存储。

## 构建验证

```bash
cd backend
pnpm build
```

```bash
cd frontend
```
