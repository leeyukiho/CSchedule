# 运行构建部署

## Ubuntu 24 后端服务器部署

安装基础依赖：

```bash
sudo apt update
sudo apt install -y curl git nginx postgresql postgresql-contrib certbot python3-certbot-nginx
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
sudo npm install -g pnpm pm2
```

创建 PostgreSQL 数据库和用户：

```bash
sudo -u postgres psql
CREATE USER cschedule WITH PASSWORD '改成强密码'
CREATE DATABASE cschedule OWNER cschedule;
\q
```

只拉取仓库里的 `backend` 目录并配置后端环境变量：

```bash
git clone --filter=blob:none --sparse https://github.com/leeyukiho/CSchedule.git /opt/cschedule
cd /opt/cschedule
git sparse-checkout set backend
cd /opt/cschedule/backend
cp .env.example .env
nano .env
```

生产环境 `.env` 主要改这些：

```bash
NODE_ENV=production
PORT=3000
DATABASE_URL=postgresql://cschedule:改成强密码@localhost:5432/cschedule?schema=public
CORS_ORIGIN=https://servicewechat.com,https://developers.weixin.qq.com,http://localhost:10086
ADMIN_API_KEY=改成随机长字符串
```

安装、初始化数据库并构建：

```bash
cd /opt/cschedule/backend
pnpm install
pnpm prisma generate
pnpm prisma migrate deploy
pnpm seed:schools
pnpm build
```

用 PM2 常驻运行：

```bash
pm2 start "pnpm start" --name cschedule-backend
pm2 save
pm2 startup
```

先把域名 DNS 解析到服务器公网 IP，例如 `api.example.com`。

Nginx 先配置 HTTP 反代到后端：

```bash
sudo nano /etc/nginx/sites-available/cschedule-backend
```

写入配置，域名改成自己的：

```nginx
server {
  listen 80;
  server_name api.example.com;

  location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
```

启用 Nginx 配置并申请 HTTPS 证书：

```bash
sudo ln -s /etc/nginx/sites-available/cschedule-backend /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
sudo certbot --nginx -d api.example.com
sudo certbot renew --dry-run
```

开放防火墙和查看日志：

```bash
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
pm2 logs cschedule-backend
```

微信开发者工具访问后端：

```bash
cd frontend
$env:TARO_APP_API_BASE_URL="https://api.example.com/api/v1"
pnpm dev:weapp
```

如果用真实小程序 AppID，去微信公众平台配置：

```bash
开发管理 -> 开发设置 -> 服务器域名 -> request合法域名
添加：https://api.example.com
```

开发阶段也可以在微信开发者工具里临时勾选：

```bash
详情 -> 本地设置 -> 不校验合法域名、web-view、TLS 版本以及 HTTPS 证书
```

正式体验版、审核版、线上版必须使用 HTTPS 证书，并配置 `request合法域名`。

## 数据库初始化

服务器第一次启动后端前执行，会运行 `backend/prisma/migrations` 里的所有迁移：

```bash
cd backend
cp .env.example .env
pnpm install
pnpm prisma generate
pnpm prisma migrate deploy
pnpm seed:schools
```

## 数据库更新

服务器更新代码后执行：

```bash
cd backend
pnpm install
pnpm prisma generate
pnpm prisma migrate deploy
pnpm build
pm2 restart cschedule-backend
```

本地开发改了 Prisma schema 或数据库结构后，可以执行：

```bash
cd backend
pnpm prisma db push
pnpm prisma generate
pnpm build
```

服务器不要用 `pnpm prisma migrate dev`，生产环境只用 `pnpm prisma migrate deploy`。

需要创建新的迁移文件时，在本地开发环境执行：

```bash
cd backend
pnpm prisma migrate dev
```

## 数据库问题

查看本地数据库：

```bash
cd backend
pnpm prisma studio
```

如果 `prisma generate` 报 `EPERM query_engine-windows.dll.node`，先关闭后端服务、Prisma Studio、微信开发者工具，再重跑。

## 用户导入、绑定、同步

涉及这些功能时注意：

- 用 `schoolId + providerId + 学号` 识别同一个学生。
- 不保存明文学号，只保存 `studentNoHash`。
- 重复导入必须复用同一个 `bindingId`。
- 临时 `bindingId` 解析到学号后，要合并到稳定 `bindingId`。
- 前端收到新 `bindingId` 后，要更新本地存储。
