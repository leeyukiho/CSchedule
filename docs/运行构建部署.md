# 运行构建部署

## 数据库初始化

第一次启动后端前执行：

```bash
cd backend
cp .env.example .env
pnpm install
pnpm prisma generate
pnpm prisma db push
pnpm seed:schools
```

## 数据库更新

改了 Prisma schema 或数据库结构后执行：

```bash
cd backend
pnpm prisma db push
pnpm prisma generate
pnpm build
```

不要直接用 `pnpm prisma migrate dev`。如果提示 drift、reset、All data will be lost，取消后改用 `pnpm prisma db push`。

只有空数据库，或确认数据可以重建时，才执行：

```bash
cd backend
pnpm prisma migrate dev
```

## 数据库问题

查看本地数据库：

```bash
cd backend
pnpm prisma studio
```

如果 `prisma generate` 报 `EPERM query_engine-windows.dll.node`，先关闭后端服务、Prisma Studio、微信开发者工具，再重跑。

## 后端运行

```bash
cd backend
pnpm install
pnpm prisma generate
pnpm build
pnpm dev
```

## 后端更新

```bash
cd backend
pnpm install
pnpm prisma generate
pnpm build
pnpm start
```

## 前端运行

```bash
cd frontend
pnpm install
pnpm dev:weapp
```

## 前端更新

```bash
cd frontend
pnpm install
pnpm build:weapp
```

## 后端地址

前端统一使用：

```bash
TARO_APP_API_BASE_URL
```

本地默认地址：

```bash
http://localhost:3000/api/v1
```

生产地址等云数据库和生产后端配置好后再启用。

## 用户导入、绑定、同步

涉及这些功能时注意：

- 用 `schoolId + providerId + 学号` 识别同一个学生。
- 不保存明文学号，只保存 `studentNoHash`。
- 重复导入必须复用同一个 `bindingId`。
- 临时 `bindingId` 解析到学号后，要合并到稳定 `bindingId`。
- 前端收到新 `bindingId` 后，要更新本地存储。
