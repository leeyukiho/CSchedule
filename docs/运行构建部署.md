# 运行构建部署

## 服务器

### 安装依赖

```bash
sudo apt update
sudo apt install -y curl git nginx postgresql postgresql-contrib certbot python3-certbot-nginx
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
sudo npm install -g pnpm pm2
```

### 创建数据库

```bash
sudo -u postgres psql
CREATE USER cschedule WITH PASSWORD '改成强密码';
CREATE DATABASE cschedule OWNER cschedule;
\q
```

### 拉取后端

```bash
git clone --filter=blob:none --sparse https://github.com/leeyukiho/CSchedule.git /opt/cschedule
cd /opt/cschedule
git sparse-checkout set backend
cd /opt/cschedule/backend
cp .env.example .env
nano .env
```

`.env` 主要改：

```bash
NODE_ENV=production
PORT=3000
DATABASE_URL=postgresql://cschedule:改成强密码@localhost:5432/cschedule?schema=public
CORS_ORIGIN=https://servicewechat.com,https://developers.weixin.qq.com,http://localhost:10086
ADMIN_API_KEY=改成随机长字符串
SYNC_ACCOUNT_COOLDOWN_MS=300000
```

### 初始化并构建

```bash
cd /opt/cschedule/backend
pnpm install
pnpm prisma generate
pnpm prisma migrate deploy
pnpm seed:schools
pnpm build
```

### 后端运行

```bash
pm2 start "pnpm start" --name cschedule-backend
pm2 save
pm2 startup
```

### 后端更新

```bash
cd /opt/cschedule/backend
git pull
pnpm install
pnpm prisma generate
pnpm prisma migrate deploy
pnpm build
pm2 restart cschedule-backend
```

### Nginx 配置

```bash
sudo nano /etc/nginx/sites-available/cschedule-backend
```

域名改成自己的：

```nginx
server {
  listen 80;
  server_name api.example.com;

  location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
```

### HTTPS 证书

```bash
sudo ln -s /etc/nginx/sites-available/cschedule-backend /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
sudo certbot --nginx -d api.example.com
sudo certbot renew --dry-run
```

### 防火墙和日志

```bash
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'
sudo ufw enable
pm2 logs cschedule-backend
```

### CloudBase 云函数

项目根目录已包含 `cloudfunctions/parseRawPayload` 和 `cloudfunctions/runProviderSync`。

```bash
cloudbase fn deploy parseRawPayload --dir cloudfunctions/parseRawPayload
cloudbase fn deploy runProviderSync --dir cloudfunctions/runProviderSync
```

如果需要本地运行函数，先执行 `cloudbase login`，然后使用：

```bash
cloudbase fn run --path cloudfunctions/parseRawPayload --params "{\"schoolId\":\"demo\",\"providerId\":\"demo\",\"target\":\"course\",\"contentType\":\"json\",\"payload\":{\"courses\":[]}}"
```

前端启用云函数 parser：

```bash
$env:TARO_APP_CLOUDBASE_ENV_ID="你的环境ID"
$env:TARO_APP_CLOUD_PARSER_FUNCTION="parseRawPayload"
pnpm build:weapp
```

如果使用 HTTP 触发器，配置 `TARO_APP_CLOUD_PARSER_URL`，前端会优先请求该地址。
