# 登录与数据获取方式

登录和数据获取是两件事，不能混在同一个概念里。

## LoginMode

`LoginMode` 只表示用户如何完成学校认证。

```ts
type LoginMode =
  | 'direct_password'
  | 'password_captcha'
  | 'cas_simple'
  | 'cas_webview'
  | 'oauth_webview'
  | 'qrcode'
```

示例：

```text
WTBU: direct_password
华夏理工 jwglxt: direct_password
广西医科大学: cas_webview
```

不要使用 `custom` 作为登录模式。教务系统类型应放在 `eduSystemType`，例如 `zf_jwglxt`、`eams`、`cas`。

## DataAccessMode

`DataAccessMode` 只表示认证后数据如何进入后端并写入缓存。

```ts
type DataAccessMode =
  | 'server_session'
  | 'webview_client_fetch'
  | 'session_import'
  | 'manual_import'
```

| 模式 | 含义 | 第一版要求 |
| --- | --- | --- |
| `server_session` | 后端登录学校系统后直接抓取数据 | 可用于无验证码或可稳定直登的学校 |
| `webview_client_fetch` | 用户在 WebView 完成认证后，前端同源获取 raw payload 并上传后端 | 用于复杂验证码、CAS、强风控学校 |
| `session_import` | 从 WebView 导入短期 Cookie/Token 后由后端抓取 | 先作为能力枚举保留，不默认承诺可用 |
| `manual_import` | 用户上传或粘贴导出的课表数据 | 后续补充 |

同一个学校可以配置多个数据获取方式，但每个功能只走自己需要的最短路径。课表同步只处理课表，不顺手抓成绩、考试和个人信息。

## 登录上下文

登录上下文用于一次绑定或一次同步，不是长期账号状态。

```ts
interface LoginContext {
  contextId: string
  schoolId: string
  mode: LoginMode
  fields: LoginField[]
  webview?: WebviewLoginDescriptor
  expireAt: string
}
```

第一版可以把临时上下文放在内存、Redis 或 `UserSchoolBinding.authState` 中，取决于代码实现。文档不强制要求单独的 Redis 或 LoginSession 表。

要求：

- 有明确过期时间。
- 不写入账号密码、Cookie、Token 到普通日志。
- 只保存完成当前登录或同步所需的最小状态。

## 绑定状态

绑定状态保存在 `UserSchoolBinding.status`。

```text
active       有可用绑定，且最近绑定或同步成功
need_login   当前没有可用认证，需要用户重新登录或进入 WebView
cached_only  没有可复用认证，但已有可读缓存
disabled     系统禁用
unbound      已解绑
```

`cached_only` 是有效状态，不是错误。它表示用户可以正常查看已有缓存，但刷新时需要重新认证或重新进入 WebView。

## WebView 学校

有复杂验证码、滑块、短信、扫码、设备校验或强 CAS 的学校，默认走学校官方 WebView，不做验证码破解。

```text
前端打开学校官方页面
  -> 用户完成认证
  -> WebView 内获取课表 raw payload
  -> POST /bindings/:bindingId/raw-data
  -> 后端解析并写 CourseCache
  -> 返回可关闭状态
  -> 前端跳转课表页读取缓存
```

如果 WebView 关闭前已经成功写入课表缓存，绑定应是 `active` 或 `cached_only`，不能显示成“未同步”。如果没有写入缓存，再显示 `need_login` 或引导重新绑定。

## 短期登录态

当前 Prisma schema 没有单独的 `LoginSession` 表。短期登录态如需保存，第一版只在 `UserSchoolBinding.authState` 中保存必要摘要或加密后的最小状态，并通过以下字段暴露能力：

```text
sessionReusable
sessionRefreshable
sessionExpireAt
lastSessionValidatedAt
credentialSaveMode
```

不要在文档或代码中假设已经存在独立的 Session 存储。只有当后台刷新真实依赖可复用 Cookie/Token 时，才新增 `LoginSession` 表。
