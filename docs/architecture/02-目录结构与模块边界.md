# 目录结构与模块边界

## 后端

```text
backend/
  src/
    modules/
      schools/       学校列表、登录上下文、同步策略
      auth/          绑定账号、保存加密凭据、接收云函数首次导入结果
      sync/          同步任务、排队、限流、云函数调用、缓存写入
      raw-data/      WebView/manual 结构化数据导入
      providers/     provider 元数据注册和展示配置
      timetable/     课表缓存读取
      features/      成绩、考试、资料缓存读取
      accounts/      学生账号和身份绑定
      admin/         管理接口
    common/
      crypto/
      prisma/
```

## 云函数

```text
cloudfunctions/
  syncWtbu/
  sync<School>/
```

学校外网访问、登录和解析只放在学校专用云函数里。

## Provider 模块边界

`providers/` 只做：

- 注册 `SchoolProvider`。
- 提供学校能力和展示配置。
- 提供凭据保存策略。

`providers/` 不做：

- 不登录学校系统。
- 不创建 HTTP client。
- 不解析学校 HTML。
- 不写数据库。

## Sync 模块边界

`sync/` 做：

- 创建 `SyncRecord`。
- active job 去重。
- 全局和学校级并发控制。
- 解密 `password_vault` 凭据。
- 调用 `CloudCredentialSyncService`。
- 写入 `CourseCache` / `FeatureCache`。

`sync/` 不做学校专用抓取解析。

## Raw Data 模块边界

`raw-data/` 只接收前端已经拿到的结构化 JSON，并写缓存。它不代理大体积 raw payload 到云函数，也不执行学校登录。
