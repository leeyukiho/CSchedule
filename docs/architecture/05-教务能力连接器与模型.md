# 教务能力连接器与模型

所有教务能力都遵循同一条边界：获取 raw payload、解析、标准化。不同能力的差异只在 raw 格式和统一模型。

## FeatureConnector

```ts
interface FeatureConnector<RawPayload, ParsedModel> {
  target: DataTarget
  supportedAccessModes: DataAccessMode[]
  fetchRaw?(auth: ProviderAuthState, query: FeatureQuery): Promise<RawPayload>
  parse(raw: RawPayload, context: FeatureParseContext): Promise<ParsedModel[]>
}
```

`ProviderAuthState` 是 Provider 内部认证状态，可以来自本次登录、`UserSchoolBinding.authState` 或 WebView 上传上下文。当前文档不要求独立 `LoginSession` 表。

使用规则：

- `server_session`：后端完成认证后调用 `fetchRaw`，再调用 `parse`。
- `webview_client_fetch`：前端上传 raw payload，后端跳过 `fetchRaw`，只调用 `parse`。
- `session_import`：导入短期登录态并验证后，才可等同 `server_session`。
- `manual_import`：用户上传文件或文本，后端直接调用 parser。

## CourseConnector

```ts
interface CourseConnector {
  target: 'course'
  supportedAccessModes: DataAccessMode[]
  fetchRawCourse?(auth: ProviderAuthState, query: CourseQuery): Promise<RawCoursePayload>
  parse(raw: RawCoursePayload, context: CourseParseContext): Promise<Course[]>
}
```

课表是第一版核心能力，必须能写入 `CourseCache`。成绩、考试、个人信息在未实现前只保留能力声明，不参与课表同步流程。

## Parser

Parser 只负责纯解析：

- 不访问数据库。
- 不做用户鉴权。
- 不发网络请求。
- 不创建同步任务。
- 不读取全局配置，必要配置从 context 传入。

## Course

```ts
interface Course {
  id?: string
  name: string
  teacher?: string
  classroom?: string
  weekday: number
  startSection: number
  endSection: number
  sections?: number[]
  weeks: number[]
  rawWeeks?: string
  campus?: string
  remark?: string
  source?: unknown
}
```

要求：

- `weekday` 统一为 `1-7`。
- `startSection` 和 `endSection` 必须能映射真实上课时间。
- `weeks` 是实际周次数组。
- 特殊课程不要伪造星期和节次。
- 调试信息不得泄露账号、密码、Cookie、Token。

## SectionTime

不同学校、校区、作息可能不同，节次时间应来自学校配置或缓存。

```ts
interface SectionTime {
  section: number
  startTime: string
  endTime: string
}
```

前端展示时间：

```text
Course.startSection / Course.endSection
  -> sectionTimesJson
  -> 真实开始和结束时间
```
