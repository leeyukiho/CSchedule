# 教务能力与缓存模型

教务能力统一落到缓存模型，而不是后端 connector。

## 数据入口

- `cloud_worker`: 学校专用云函数返回标准 `cacheResults`。
- `webview_client_fetch`: 前端上传结构化 JSON。
- `manual_import`: 用户手动导入结构化数据。

后端只做校验、标准化和写缓存。

## Course Cache

课程缓存写入字段：

```ts
{
  termId?: string
  courses: Course[]
  terms?: unknown[]
  sectionTimes?: unknown[]
  sourceHash?: string
  syncedAt?: string
}
```

`Course`：

```ts
interface Course {
  id?: string
  name: string
  teacher?: string
  location?: string
  classroom?: string
  weekday: number
  sections?: number[]
  startSection?: number
  endSection?: number
  weeks: number[]
  rawWeeks?: string
  campus?: string
  remark?: string
  source?: unknown
}
```

## Feature Cache

成绩、考试、资料统一写入 `FeatureCache`：

```ts
{
  target: 'score' | 'exam' | 'profile'
  termId?: string
  data: unknown
  meta?: Record<string, unknown>
  sourceHash?: string
  syncedAt?: string
}
```

`meta.source` 对云函数结果使用 `cloud_worker`。

## 云函数职责

云函数负责：

- 登录学校系统。
- 按 target 抓取必要数据。
- 解析并标准化为缓存结构。
- 返回 `cacheResults`。

云函数不负责：

- 创建用户账号。
- 写数据库。
- 创建同步任务。
- 读取后端全局状态。

## 后端职责

后端负责：

- 凭据加密保存和解密。
- 同步任务排队、去重和限流。
- 调用学校专用云函数。
- 标准化 course 字段。
- 写入 `CourseCache` / `FeatureCache`。
