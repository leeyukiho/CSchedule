# 保存密码与自动同步专项授权协议

适用范围：本协议适用于用户在绑定学校账号、导入课表或同步设置中主动勾选“保存密码”“加密保存学校账号密码，用于自动同步”或类似选项后的凭据保存、使用、同步和管理全过程。

## 特别提示

学校账号密码、统一身份认证密码、验证码、Cookie、Token、Session、登录态及等效认证信息属于敏感或高风险信息。保存密码和自动同步功能可以减少用户重复登录，但也可能带来账号安全、学校风控、学校规则、数据安全和第三方系统限制等风险。

用户未勾选保存密码的，仍可在学校支持的情况下使用一次性导入、WebView 导入、手动导入或重新输入密码同步等基础功能。用户勾选保存密码、点击确认、提交账号密码或继续使用自动同步功能，即表示用户已单独、明确授权平台按照本协议处理用户学校账号密码及相关凭据。

除非平台在特定学校页面以明确文字标注“官方合作”“学校授权”或类似表述，保存密码和自动同步功能不代表学校、教务处、信息化部门、统一身份认证系统或其他第三方主体授权、认可、背书或担保平台。

如学校明示禁止第三方工具保存密码、自动登录、自动同步、页面抓取、接口调用或类似行为，用户应立即停止使用本功能并删除已保存密码。平台有权根据学校要求、公开规则、投诉反馈或平台合理判断，随时限制、暂停或关闭相关学校的保存密码或自动同步能力。

## 一、专项授权范围

用户授权平台在实现保存密码、首次导入、手动同步、自动同步、同步失败恢复、账号绑定校验、安全风控和服务维护所必需的范围内，处理下列信息：

1. 学校账号、学号、统一身份认证账号、手机号、邮箱或其他登录标识。
2. 学校密码、统一身份认证密码或其他由用户主动提交的登录密码。
3. 登录上下文、学校标识、provider 标识、账号绑定关系、同步目标、同步状态、错误码、同步时间、数据哈希、失败记录、撤回状态和必要技术日志。
4. 学校系统返回的课表、学期、节次时间、教室、教师、考试、成绩、个人资料摘要及用户启用或学校支持的其他数据。
5. 为自动同步所必需的加密凭据、密钥标识、保存时间、最后使用时间、授权状态、同步策略和风控状态。

用户授权平台通过本服务客户端、后端服务、云函数、托管服务、数据库、缓存、队列、日志系统、监控系统和受委托的基础设施服务处理上述信息。

## 二、使用目的

平台保存和使用用户学校账号密码的目的限于：

1. 完成用户主动发起的首次导入、重新登录或重新绑定。
2. 在用户主动点击同步时，代用户向学校系统或学校相关认证系统请求最新数据。
3. 在用户启用或平台支持的低频自动同步范围内，定期刷新课表及相关数据。
4. 识别用户本人学校身份，合并重复绑定账号，维护账号与学校数据的对应关系。
5. 在同步失败、登录态失效、学校接口变化、密码错误或数据异常时进行必要的错误识别、提示和恢复。
6. 进行安全风控、限流、熔断、反滥用、日志审计、故障排查、服务稳定性维护和学校适配优化。

平台不会将用户学校账号密码用于广告投放、画像营销、出售、出租、公开披露或与本协议无关的用途，法律法规另有规定、监管要求、司法行政机关要求或为保护用户、平台、学校、第三方合法权益所必需的除外。

## 三、保存方式和安全措施

1. 平台原则上不保存明文学校密码。用户选择保存密码后，平台将对账号密码进行加密存储。
2. 平台将根据实际情况采取权限控制、密钥管理、日志过滤、访问审计、保留期限限制、安全监测、异常告警等合理安全措施。
3. 平台不会在普通业务日志、错误日志、调试输出、公开文档或客服沟通中主动展示用户明文密码。
4. 平台将尽合理努力限制能够接触加密凭据、密钥、同步日志和学校数据的人员、系统和服务范围。
5. 平台可能使用云服务商、数据库、对象存储、云函数、消息队列、监控系统、CDN、安全服务或其他基础设施承载保存密码和自动同步功能。相关服务提供方仅在技术服务所必需范围内处理数据。
6. 用户理解，任何互联网服务均无法保证绝对安全。尽管平台采取合理措施，仍可能因学校系统、第三方服务、网络攻击、系统漏洞、用户设备风险、用户泄密、不可抗力或其他原因发生安全事件。

## 四、自动同步方式和限制

1. 自动同步可能由平台后端、云函数或其他受控技术组件在用户授权范围内，使用已加密保存的凭据登录或访问学校系统，获取用户启用的数据。
2. 自动同步频率、时间、目标数据、并发限制、失败重试、冷却时间、熔断策略和可用学校，由平台根据学校支持情况、系统负载、学校风控、运营策略和安全要求动态调整。
3. 平台可默认仅同步课表，也可在用户启用或学校支持时同步考试、成绩、个人资料摘要等数据。具体以页面展示、学校配置和用户设置为准。
4. 平台不承诺自动同步一定成功，不承诺定时、实时或持续可用，不承诺学校系统不会触发验证码、风控、限流、异常登录提醒、账号保护或其他限制。
5. 学校系统要求验证码、短信验证、扫码验证、二次认证、密码重置、人工确认或其他用户交互时，平台可能暂停自动同步并要求用户重新登录或改用手动导入。
6. 平台可为保护学校系统稳定和平台安全，对自动同步设置学校级、账号级、目标数据级、设备级、IP 级或时间段级限制。

## 五、用户使用范围和承诺

用户确认并承诺：

1. 用户是所提交学校账号的合法使用人，或已取得合法、充分、有效的授权。
2. 用户保存密码和自动同步仅用于本人个人学习、生活、课程管理和时间提醒。
3. 用户提交的账号密码真实、准确、合法、有效，不属于他人账号、共享账号、测试账号、盗用账号或无权使用账号。
4. 用户使用保存密码和自动同步功能不违反法律法规、学校规则、第三方认证规则、学生管理规定、网络安全要求或其他适用规则。
5. 用户不会利用保存密码或自动同步功能进行批量账号同步、代他人同步、商业化抓取、数据倒卖、模型训练、接口转售、攻击测试、绕过验证、规避风控或其他不当行为。
6. 用户应及时更新错误密码、停止异常同步、处理学校安全提醒，并在账号遗失、密码泄露、设备丢失或发现异常时立即修改学校密码并撤回授权。

1. 用户可以通过平台提供的功能或平台公布的联系方式，撤回保存密码授权、关闭自动同步、解除学校绑定、删除相关数据或注销账号。
